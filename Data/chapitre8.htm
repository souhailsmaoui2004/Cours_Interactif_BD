1- Introduction

Le langage de Manipulation de Données (LMD) permet la consultation et la mise à jour des objets crées par le langage de définition de données. La mise à jour englobe l'insertion de nouvelles données, la modification et la suppression de données existantes.

2- Insertion de données

L'insertion consiste à ajouter des nouvelles lignes dans une table. Elle peut affecter toutes les colonnes d'une tble ou seulement un sous-ensemble de ces colonnes, les autres prendront automatiquement des valeurs nulles.

On distingiue entre 2 formes por la commande d'insertion :

2.1- Insertion des valeurs pour la totalité des colonnes de la table

- Syntaxe :

INSERT INTO nom_table VALUES [colonne1, colonne2,...)] VALUES ( vaeur1,valeur2,...);

Les valeurs des colonnes de type caractère ou chaîne de caractères doivent appara^tre entre 2 cotes (''). Si la valeur est nulle, on doit mettre 2 cotes successives(sans blanc). Si le type est numérique et la valeur est nulle, il suffit de la dépasser mais il ne faut mettre la liste des champs qui devient indispensable sans mettre le champ de dont la valeur sea nulle.

- Exemple1 :

INSERT INTO client VALUES (100,'mohamed',98777666,'rue ezzouhour 102',3093,'Sfax');

- Exemple2 :

INSERT INTO client(numcl,nomcl) VALUES (100,'karima turki');

2.2- Insertion à travers la copie des valeurs des colonnes d'une autre table

- Syntaxe :

INSERT INTO nom_table [(colonne1,colonne2,...)] requête

- Exemple :

INSERT INTO client (numcl,nomcl,tel,adr) SELECT * FROM client_mahdia;

3- Modification des Données

L'opération de modification de données consiste à mettre à jour les colonnes d'une table par de nouvelles valeurs.

- Syntaxe :

UPDATE nom_table

SET col1= val1,...,coln =valn WHERE condition;

—> Il n'est pas possible de mettre à jour plus qu'une table à la fois.

—> Les valeurs peuvnt être des constantes, des expresions ou des réultats de sous-requêtes.

- Exemple :

UPDATE client SET adr_clt ='10,cité kantaoui' WHERE nom ='mohamed';

4- Suppression de données

La suppression des données consiste à supprimer une ou plusieurs lignes d'une table.

- Syntaxe :

DELETE FROM nom_table WHERE condition;

- Exemple :

DELETE FROM client WHERE ville = 'Tunis';

5- Consultation de Données

 

La consultation se fait par la syntaxe suivante :

 

SELECT liste_d'attributs FROM liste_de_tables

[WHERE condition ]

[[START WITH condition] CONNECT BY condition ]

[GROUP BY list_attributs] [HAVING condition]

[UNION /UNION ALL /INTERSECT/MINUS] commande SELECT ]

[ORDER BY [expre/position] [ASC/DESC];

La clause SELECT et FROM sont obligatoires et permettant de spécifier la liste des attributs à extraire et la liste de tables à partir desquelles les données seront extraites.

Toutes les autres clauses sont optionnelles et servent à raffiner la sélection et offrir d'autres services.

- L'ordre des colonnes est celui spécifié dans la liste de sélection. Si l'utilisateur désire lister la totalité des colonnes de la table, il peut utiliser le caractère génerique *.

- Si on souhaite avoir toutes les lignes de la table, on ne met pas la clause WHERE.

- Chaque fois qu'une reqûete d'interrogation est exécutée, les noms des colonnes définies dans la structure de la table apparaissent comme titres de colonnes. On peut modifier ces noms de colonnes à l'affichage uniquement en ajoutant des ALIAS. La commande devient comme suit:

 

SELECT col1[alias1], col2 [alias2],...coln[aliasn]

FROM nom_table [alias]

......

5.1- Cnsultation Simple

La syntaxe est la suivante : SELECT list_attr FROM liste_table;

- Exemple :

SELECT * FROM client;

5.2- Consultation avec qualification

Elle se fait en ajoutant la clause WHERE et en spécifiant une ou plusieurs conditions. On peut dans une condition utiliser les opérateurs de comparaison qui sont offerts par SQL tels que =,<>,<,>,<=,>=.

La comparaison concerne bien les valeurs numériques,les chaînes de caractères et les dates.

La condition est composée généralement de 3 termes :

* Un nom de colonne

* Un opérateur de comparaison

* et une costante ou une colonne ou une liste de valeur.

En plus des opérateurs de compraison déjà cités ORACLE dispose d'un certain nombre d'opérateurs spécifiques dont principalement sont :

* IS NULL : permet de tester si le contenu d'une colonne est une valeur nulle ( indéfinie).

* IN (liste de valeurs) : permet de tester si le contenu d'une colonne coincide avec l'une des valeurs de la liste.

* BETWEEN V1 AND V2 : permet de tester si le contenu d'une colonne est compris entre les valeurs V1 et V2.

* LIKE chaîne générique : le prédicat LIKE permet de réaliser une comparaison entre la valeur de la colonne et celle de la chaîne en utilisant des caractères génériques de substitution. Les caractères génériques de substitution sont :

 

% : substitue 0 o plusieurs caractères pour une chaîne.

- : remplace un seul caractère.

Tous les opérateurs spécifiques peuvent être mis sous forme négative en les faisant précéder par l'opérateur de négation NOT.

- Exemples :

Soit la table Produit et client définies par :

produit(codep,designation,qtest,prixu)

client(numcl,nom,ville)

1) Lister toous les produits dont la quantité en stock est inférieure à 10.

 

SELECT * FROM produit WHERE qtest <10;

2) Sélectionner la désignation et le prix des produits dont le prix unitaire est compris entre 5 et 10.

 

SELECT designation , prixu FROM produit WHERE prixu BETWEEN 5 AND 10;

3)Sélectionner les clients de Sfax et de Tunis

 

SELECT * FROM client WHERE ville in ('Sfax','Tunis');

4)Sélectionner les noms de clients qui commençent par la lettre T.

 

SELECT nom FROM client WHERE nom LIKE 'T%';

5) Sélectionner les noms de clients dont la deuxième lettre est R.

 

SELECT nom FROM client WHERE nom like '-R%';

5.3- Les Expressions Arithmétiques

Dans la clause SELECT et WHERE, on peut utiliser des expressions arithmétiques et des fonctions telles que :

* ABS(n) : permet de calculer la valeur absolue de n.

* CEIL(n) : permet d'avoir le plus petit entier supérieur ou égal à n.

* FLOOR(n) : permet d'avoir la partie entière de n.

* MOD(n,m) : permet d'avoir le reste de la division entière de m par n.

* ROUND(m,n) : permet d'arrondir la valeur n à m décimal.

* POWER(m,n) : permet d'avoir m puissance n.

* SIGN(n) : donne -1 si n < 0, 0 si n =0 et 1 si n> 1.

* SQRT(n) : permet d'avoir racine de n.

* TRUNC(m,n) : permet de tronquer la valeur après m décimales. Si n est négatif, la valeur de m est tronquée avant le point décimal.

- Exemples :

1) TRUNC(121.371,1) donne 121.3

 

TRUNC(121.371,-1) donne 120

TRUNC(121.371,-2) donne 100

2)Donner le prix de tous les produits arrondis en Dinar.

 

SELECT designation, ROUND(prixu) " prix en D" FROM produit;

5.4- Les Fonctions s'appliquant à des chaînes de caractères

* RTRIM(ch) : supprime l'espace à la fin de la chaîne à l'opposé de LTRIM.

* RPAD(ch,n) : ajoute n espaces à la fin de la chaîne à l'opposé de LPAD.

* INITCAP(ch) : met en majuscule la première lettre de chaque mot de la chaîne.

* INSTR(ch1,ch2,[n[,m]]) : cherche la sous-chaîne ch2 dans la chaîne ch1 à partir de la position n et si l'on souhaite à partir de la m ème occurence de ch2 dan ch1. n et m sont optionnels et ont pour valeur par défaut 1.

* LENGTH(ch) : renvoie la longueur d'une chaîne.

* LOWER (ch) : transform la chaîne ch en miniscule à l'opposé de UPPER.

* SUBSTR(ch,m,n) : extraire la sous-chaîne de ch qui commence à partir du caractère à la position m et de longueur n.

* TRANSLATE(ch,ch1,ch2) : permet de transformer dans la chaîne ch toutes les occurences de ch1 par ch2.

* REPLACE(chaine,ch1 [,ch2]) : remplace une chaîne par une autre dans une colonne. Si on ne met pas ch2, ch1 va être emplacée par un vide.

* ch1 ||ch2 : concatène les deux chaînes.

- Exemple:

Soit la table suivante :

Magasin(num,nom,mnager)

num nom manager

m2 Sousse dhouib

SELECT SUBSTR(manager 2,3) FROM Magasin;

donne : rki , ouib

5.7 - Les expressions Aregats( ou fonctions de groupe)

ORACLE dispose d'un ensemble de fonctions appelées fonctions agrégats qui s'appliquent à un ensemble de données :

* AVG : permet de donner la moyenne arithmétique d'un ensemble donné.

* COUNT: permet d'avoir le nombre d'occurences des enregistrements.

* MAX : permet d'avoir la valeur maximale dans une colonne.

* MIN : permet d'avoir la valeur minimale dans une colonne.

* SUM : permet d'avoir la somme des élements.

* STDDEV : permet d'avoir l'ecart -type.

* VARIANCE : permet d'avoir la variance.

* DISTINCT : pour ne pas prendre que des valeurs distinctes.

* ALL : pour insister de prendre toutes les valeurs qui existent.

- Chacune de ces fonctions a comme argument un nom de colonne ou une expression arithmétique. Elles ignorent les valeurs nulles et par défaut prennent les valeurs multiples pour des valeurs différentes.

- La fonction COUNT peut prendre comme argument la caractère génerique * ce qui veut dire que nous voulons connaître le nombre de ligne séletionnées.

- On ne peut pas mettre une fonction de groupe après la clause WHERE.

- Exemple :

Donner le nombre de clients de la ville de Sfax dont le numéro de téléphone commence par 221.

 

SELECT COUNT(*) FROM client WHERE ville='Sfax' and trunc(tel,)3) =221000;

5.8- Classification des Résultats

Il est possible de regrouper ( classifier) les résultats obtenus permettant d'avoir ensemble les lignes ayant une même vaeur pour la colonne de classification. La clause qui permet la classification est: GROUP BY expression [HAVING condition].

- Exemple :

1) Trouver la somme des quantités commandées par produit et par client qui sont superieures à 10.

 

SELECT nump,numclt,sum(qte_cde) FROM commande

GROUP BY nump,numclt

HAVING sum(qte_cde) > 10;

5.9- Tri des Résultats

Pour obtenir un résultat trié, il suffit d'ajouter à la requête SQL la clause ORDER BY expression [ASC/DESC].

Si on ne spécifie pas ASC ou DESC, le tri sera croissant par défaut.

- Exemple :

Donner la liste des produits par ordre de prix unitaire croissant et ceux ayant le même prix seront triés par ordre alphabétique décroissant.

 

SELECT designation,prixu from produit ORDER BY prixu,designation desc;

5.10- Les requêtes imbriquées

La requête imbriquée figure lorsqu'une requête apparaît dans la clause WHERE de la requête principale.

- Exemple :

1) Trouver la désignation des produits dont le prix unitaire est égal à celui des chaises.

 

SELECT designation,prixu FROM produit WHERE prixu = (SELECT prixu FROM produit WHERE designation ='chaise');

2) Trouver les produits dont le prix est superieur au prix moyen

 

SELECT designation, prixu FROM produit WHERE prixu > (SELECT AVG(prixu) FROM produit);

5.11- Les Opérateurs Ensemblistes

- Syntaxe :

La forme génerale de ces opérateurs est :

 

Requête1 opérateur Requête2

- Remarques:

Pour tous les opérateurs, il faut:

* Utiliser le même nombre de variables avec les mêmes types dans les 2 requêtes.

* Les doublons sont automatiquement éliminés.

* Les titres des colonnes résultat sont ceux de la 1 ère requête.

* La clause ORDER BY ne fait plus réference aux noms de colonnes, mais à leurs numéros.

5.11.1- L'Union

C'est une opération qui porte sur 2 relations R1 et R2 consistant à construire une relation de même schéma ayant pour tuples ceux appartenant à R1 ou R2 ou aux 2 relations.

La forme génerale pour cette relation est :

 

Requête1 UNION requête2

- Exemple :

Soit la relation suivante :

Etudiant(mat_et,nom_et,pre_et,ville,classe,groupe)

Donner les noms des etudiants du groupe G1 et ceux du groupe G2

 

SELECT nom_et, pre_et FROM etudiant WHERE groupe ='G1'

 

UNION

SELECT nom_et, pre_et FROM etudiant WHERE groupe ='G2'

5.11.2- L'Intersection

C'est une opération portant sur 2 relations de même schéma ayant pour tuples ceux appartenant à R1 et R2 en même temps.

La forme génerale pour cette relation est :

 

Requête1 INTERSECT requête2

- Exemple :

Donner les noms des étudiants de la classe IG4 et qui habitent à Sfax.

 

SELECT nom_et FROM etudiant WHERE classe ='IG4'

 

INTERSECT

SELECT nom_et FROM etudiant WHERE ville='Sfax';

5.11.3- La Différence

C'est une opération portant sur 2 relations de même schéma R1 et R2 consistant à construire une relation de même schéma ayant pour tuples ceux appartenant à R1 et n'appartenant pas à R2.

La forme génerale pour cette relation est :

 

Requête1 MINUS requête2

- Exemple :

Donner les Noms des etudiants de l'ISET sauf ceux de la classe IG4.

 

SELECT nom_et, pre_et FROM etudiant

 

MINUS

SELECT nom_et, pre_et FROM etudiant WHERE classe ='IG4';

5.11.4- La Jointure

Jointure avec SQL2:

C'et une opération portant sur 2 relations R1 et R2. Elle produit une relation R3 dont le Schéma est la juxtaposition de deux Schéms R1 et R2 et contenant l'ensemble des tuples obtenus par concaténation des tuples de R1 et R2 vérifiant une condition défini entre R1 et R2. Géneralement la condition définie est sous forme de comparaison de valeurs d'une ou de plusieurs colonnes communes à ces 2 relations et qui évidemment ont les mêmes types et les mêmes tailles.

On distingue entre 3 types de jointure :

* Jointure Simple : c'est celle déjà présentée

* Autojointure : c'et une jointure entre la table et elle même

* Jointure Externe : permet de conserver toutes les valeurs d'un attribut appartenant à R1 dans la relation résultat. De point de vue syntaxique, cette opération se formule de la même façon qu'une jointure classique sauf qu'il faut ajouter à la condition de jointure le signe (+).

- Exemple 1 : Jointure Simple

Soit les 2 relations suivantes :

Produit(codp,designation,prixu,qte_stk)

lignecom(numc,codp,qte_com)

Donner les désignations des produits dont le numéro commande est 25

SELECT designation FROM produit,lignecom WHERE produit.codp = lignecom.codp and numc =25;

- Exemple 2 : Autojointure

Donner la désignation des produits dont la quantité en stock est superieure ou égale à celle du produit 'ordinateur'.

Solution1 : Créer un synonyme de la table produit

CREATE SYNONYM P FOR PRODUIT;

SELECT P.designation FROM P,produit WHERE produit.designation ='ordinateur' AND P.qte_stk >= produit.qte_stk;

Solution2 : Cette solution consiste à utiliser une variable ALIAS pour la table produit

SELECT p2.designation FROM produit P1,produit P2 WHERE P1.des_pdt='ordinateur' AND P2.qte_stk>=P1.qte_stk;

Solution3 : Cette solution consiste à utiliser une requête imbriquée

SELECT P1.designation

FROM produit P1

WHERE P1.qte_stk >= ( SELECT qte_stk FROM produit P2 WHERE P2.designation ='ordinateur');

- Exemple 3 : La jointure Externe

Donner la désignation des produits avec éventuellement les commandes correspondantes

Solution

SELECT des_pdt,LC.numc

FROM produit P,lignecom LC

WHERE P.codp = LC.codp(+)

Jointure avec SQL3:

Sans compter l'opérateur CROSS JOIN, voici les trois syntaxes possibles de l'expression d'une jointure dans la clause FROM en SQL :
table_1 { [INNER] { LEFT | RIGHT | FULL } [OUTER] } JOIN table_2 ON predicat [...]
table_1 { [INNER] { LEFT | RIGHT | FULL } [OUTER] } JOIN table_2 USING (colonnes) [...]
table_1 NATURAL { [INNER] { LEFT | RIGHT | FULL } [OUTER] } JOIN table_2 [...]
Ces trois syntaxes diffèrent par la condition de jointure spécifiée par les clauses ON ou USING, ou implicite dans le cas d'une jointure naturelle introduite par le mot-clé NATURAL.


ON :
La clause ON correspond à la condition de jointure la plus générale. Le prédicat predicat est une expression logique de la même nature que celle de la clause WHERE décrite dans la section 4.5.7.
USING :
La clause USING est une notation abrégée correspondant à un cas particulier de la clause ON. Les deux tables, sur lesquelles portent la jointure, doivent posséder toutes les colonnes qui sont mentionnées, en les séparant par des virgules, dans la liste spécifiée entre parenthèses juste après le mot-clé USING. La condition de jointure sera l'égalité des colonnes au sein de chacune des paires de colonnes. De plus, les paires de colonnes seront fusionnées en une colonne unique dans la table résultat de la jointure. Par rapport à une jointure classique, la table résultat comportera autant de colonnes de moins que de colonnes spécifiées dans la liste de la clause USING.
NATURAL :
Il s'agit d'une notation abrégée de la clause USING dans laquelle la liste de colonnes est implicite et correspond à la liste des colonnes communes aux deux tables participant à la jointure. Tout comme dans le cas de la clause USING, les colonnes communes n'apparaissent qu'une fois dans la table résultat.
INNER et OUTER :
Les mots-clé INNER et OUTER permettent de préciser s'il s'agit d'une jointure interne ou externe. INNER et OUTER sont toujours optionnels. En effet, le comportement par défaut est celui de la jointure interne (INNER) et les mots clefs LEFT, RIGHT et FULL impliquent forcément une jointure externe (OUTER).
INNER JOIN :
La table résultat est constituée de toutes les juxtapositions possibles d'une ligne de la table table_1 avec une ligne de la table table_2 qui satisfont la condition de jointure.
LEFT OUTER JOIN :
Dans un premier temps, une jointure interne (i.e. de type INNER JOIN) est effectuée. Ensuite, chacune des lignes de la table table_1 qui ne satisfait pas la condition de jointure avec aucune des lignes de la table table_2 (i.e. les lignes de table_1 qui n'apparaissent pas dans la table résultat de la jointure interne) est ajoutée à la table résultats. Les attributs correspondant à la table table_2, pour cette ligne, sont affectés de la valeur NULL. Ainsi, la table résultat contient au moins autant de lignes que la table table_1.
RIGHT OUTER JOIN :
Même scénario que pour l'opération de jointure de type LEFT OUTER JOIN, mais en inversant les rôles des tables table_1 et table_2.
FULL OUTER JOIN :

La jointure externe bilatérale est la combinaison des deux opérations précédentes (LEFT OUTER JOIN et RIGHT OUTER JOIN) afin que la table résultat contienne au moins une occurrence de chacune des lignes des deux tables impliquées dans l'opération de jointure.

 

 

 

 

6- Manipulation de données à travers des vues
La Notion de vue permet à un utilisateur de manipuler autrement les données sans se recourir aux tables définies dans la base.
Une vue est en fait une perception particulière des données d'une ou de plusieurs tables. Elle est stockée sous forme de requêtes de sélection dans le dictionnaire des données.
Elle permet de :
- Restreindre l'accès à certaines colonnes ou certaines lignes d'une table : l'accès se fait uniquement à travers la vue.
- Simplifier la tâche de l'utilisateur en le déchargeant de la formulation de requêtes complexes.

 

6.1- Création des Vues

La création d'une vue consiste à définir son schéma, c'est à dire son nom et ceux des colonnes qui la composent, et les données qui lui sont rattachées.
- Syntaxe :
CREATE VIEW vue [(colonne1,...,colonnen)] AS requête
[WITH CHECK OPTION];
La clause WITH CHECK OPTION permet de vérifier l'effet des opérations INSERT et UPDATE réalisée sur la vue. En effet, cette clause permet :
- D'autoriser ou de ne pas autoriser une mise à jour qui ne respectepas la condition de sélection qui définit la vue.
- D'autoriser ou d'interdire l'ajout d'une ligne qui ne satisfait pas la condition de sélection de cette vue à travers la vue.
—> Si cette clause est omise, aucune vérification n'aura lieu.
- Exemple :
Créer une vue correspondant aux clients de la ville de Tunis.
CREATE VIEW client_tunis AS SELECT idclient,nom,adresse,codepostal,tel FROM client WHERE ville='Tunis';
6.2- Suppression des Vues
La suppression d'une vue consiste à supprimer du dictionnaire de données la définition correspondante. Les données attachées de la vue ne sont pas supprimées.
- Syntaxe :
DROP VIEW nom_de_la_vue;
6.3- Modification des Structures de vues
Aucune modification n'est pas permise sur les structures des vues.
6.4- Consultation des Vues
Toutes les requêtes possibles sur les tables sont autorisées sur les vues.
6.5- Mise à jour des Vues
- Si la vue dérive de plusieurs tables ou si sa définition contient la clause GROUP BY, la clause DISTINCT ou encore une fonction de groupe, alors les opérations UPDATE, INSERT et DELETE sont interdites.
- Si un attribut de la vue est défini comme une expression, cet attribut ne peut subir d'UPDATE et d'INSERT qui sont interdites sur la vue.
- Si un attribut déclaré NOT NULL dans la table de base n'est pas repris dans la vue, aucune INSERT n'est possible sur cette vue.
6.6- Règles génerales pour l'utilisation d'une vue
- La jointure entre une vue groupée et une table est interdite.
- Les colonnes de la vue, obtenues par groupement ne peuvent en aucun cas figurer dans la clause WHERE de la requête d'interrogation.
- Des anomalies peuvent être détectées si la définition de la vue utilise DISTINCT, une opération arithmétique, une opération de concaténation de chaînes de caractères.
