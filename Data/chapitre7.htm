1- Introduction
Le langage de définition de données (LDD) permet de définir des bases de données et des objets qui les composent.
La définition d'un objet inclut géneralement sa création, sa modification et sa suppression.
Les Commandes de LDD sont:
- CREATE : création d' objets
- ALTER : modification des structures d'objets.
- DROP : suppression d'objets.

2 Création d'une Table

La création d'une table consiste à définir son nom, les colonnes qui la composent et leurs types. Elle se fait avec la commande CREATE TABLE

ORACLE permet de créer les tables de 2 façons :

- Création simple

- Création + Insertion

2.1- Creation Simple

- Syntaxe :

CREATE TABLE nom_table

(

col1 type [(taille)] [DEFAULT valeurpardefaut] [NULL/NOT NULL] [ contraintes de colonnes],

col2 type [(taille)] [DEFAULT valeurpardefaut] [NULL/NOT NULL] [ contraintes de colonnes],

colp type [(taille)] [DEFAULT valeurpardefaut] [NULL/NOT NULL] [ contraintes de colonnes],

[contrainte de tables]

);

- Définition des colonnes :

- La taille indique la valeur maximale de la longueur du champ.

- L'option NOT NULL indique que le champ doit forcément avoir une valeur.

- Les types de donnes possibles sont :

->NUMBER : désigne les nombres(positifs ou négatifs).

->NUMBER(n) : le nombre de taille spécifique n <=105.

->Number(n,m) : nombre décimal pouvant avoir au maximum n chiffres dont m après la virgule.

->CHAR(n) : chaine de caractères de taille fixe avec 1<=n<=255.

->VARCHAR2(n) : chaîne de caractères de longueur fixe avec 1<=n<=4000.

->RAW : accepte jusqu'à 2000 octets de données binaires.

->LONG RAW :données de type binaire de longueur variable allant jusqu'à 2G Octets.

->DATE: date dont le format standard est DD-MM-YY.

- EXEMPLE:

Créer une table crrespondante à l'entité client ayant comme clé le champ numero cient. Cette table comprend les champs relatifs au nom, teléphone,adresse , code postal et ville.

CREATE TABLE client

(numclt NUMBER(6),

nom VARCHAR2(15),

tel NUMBER(8) ,

adr VARCHAR2(30),

cod_post NUMBER(4),

ville char(12)

);

- Définition des Contraintes

Une contrainte d'integrité est une règle qui définit la cohérence d'une donnée ou d'un ensemble de données de la base de données.

Les contraintes définies par SQL sont :

- Non nullité des valeurs d'un attribut : l'utilisateur es obligé de saisir la valeur de l'attribut, et ce à travers le mot clé NOT NULL.

- Unicité de la valeur d'un attribut ou d'un groupe d'attributs : la valeur doit être unique et ce à travers le mot clé UNIQUE.

- Clé primaire ( un attribut ou un groupe ) : indique que l'attribut est une clé primaire et ce à tavers la syntaxe suivante :

CONSTRAINT nom_contrainte PRIMARY KEY (att1,att2,...attn);

EXEMPLE:

CREATE TABLE client

(numclt NUMBER(6) PRIMARY KEY,

nom VARCHAR2(15),

tel NUMBER(8) ,

adr VARCHAR2(30),

cod_post NUMBER(4),

ville char(12)

);

CREATE TABLE client

(numclt NUMBER(6) PRIMARY KEY,

nom VARCHAR2(15),

tel NUMBER(8) ,

adr VARCHAR2(30),

cod_post NUMBER(4),

ville char(12),

CONSTRAINT pk_client PRIMARY KEY numclt

);

- Clé étrangère :

Lorsque la clé primaire figure dans une autre table en tant qu'un attribut non clé. La syntaxe est :

CONSTRAINT nom_contrainte FOREIGN KEY (nom_att) REFERENCES nom_table (nom_att);

Il est impossible de créer une clé étrangère si la clé primaire associée n'existe pas.

EXEMPLE :

CREATE TABLE vente

( codvent NUMBER(6) PRIMARY KEY,

numclt NUMBER(6) ,

qtevnt NUMBER(8),

CONSRAINT fk_vente FOREIGN KEY (numclt) REFERENCES client (numclt));

ou

CREATE TABLE vente

( codvent NUMBER(6) PRIMARY KEY,

numclt NUMBER(6) REFERENCES client(numclt),

qtevnt NUMBER(8));

- Contrainte de valeur avec la clause CHECK : permet de limiter les valeurs possibles pour une colonne en vérifiant une certaine condition. Le contrôl se fera lors des insertions des données.

CONSTRAINT nom_contrainte CHECK( colonne condition).

La condition sur la colonne peut utiliser :

-> Un Opérateur de compraison : <,>, =, <> , <=,>=

-> La clause BETWEEN val1 AND val2

-> La clause IN ( liste des valeurs).

EXEMPLE:

CREATE TABLE vente

( codvent NUMBER(6) PRIMARY KEY,

numclt NUMBER(6) REFERENCES client(numclt),

qtevnt NUMBER(8),

CONSTRAINT ck_vente_qte CHECK (qtevnt between 0 and 10,

CONSTRAINT ck_vente_clt CHECK(numclt IN (10,11,12,13));

2.2- Création + Insertion

En plus de la définition de la structure d'une nouvelle table, cette forme permet d'y insérer des données à partir d'une ou plusieurs tables et/ou vues.

- Syntaxe

CREATE TABLE NOM8TABLE

[

col1 type [(taille)] [DEFAULT valeurpardefaut] [NULL/NOT NULL] [ contraintes de colonnes],

col2 type [(taille)] [DEFAULT valeurpardefaut] [NULL/NOT NULL] [ contraintes de colonnes],

colp type [(taille)] [DEFAULT valeurpardefaut] [NULL/NOT NULL] [ contraintes de colonnes],

[contrainte de tables]

]

AS requête;

Si la définition des colonnes est omise, ORACLE hérite les noms de colonnes de la requête. En effet la requête est une opération de sélection qui indique les colonnes sources, les tables auxquelles appartiennent et éventuellement des critères de sélection.

EXEMPLE:

Création d'une table contenant des clients de la ville de Mahdia :

CREATE TABLE clent_mahdia

AS

SELECT numclt,nom,tel,adr

FROM client

WHERE ville='MAHDIA';

3- Modification de la Structure d'une table

Les tables peuvent avoir une structure dynamique ( ajout de nouvelles colonnes et/ou modification de colonnes déjà existantes.

La suppression de colonnes n'est pas possible directement.

Les deux possibilités de modification se traduisent par deux variantes de la commande ALTER TABLE

3.1- Ajout de Nouvelles colonnes à une table

- Syntaxe :

ALTER TABLE nom_table

ADD ( col1 type [(taille)][NULL/NOT NULL],

col2 type [(taille)][NULL/NOT NULL],

colp type [(taille)][NULL/NOT NULL]);

- Exemple :

ALTER TABLE client

ADD type_clt varchar2(10);

3.2- Modification de la structure d'une colonne existante

- Syntaxe :

ALTER TABLE nom_table

MODIFY ( col1 type [(taille)][NULL/NOT NULL],

col2 type [(taille)][NULL/NOT NULL],

colp type [(taille)][NULL/NOT NULL]);

- Exemple :

ALTER TABLE client

MODIFY type_clt varchar2(15);

3.3- Ajout d'une contrainte

- Syntaxe :

ALTER TABLE nom_table

ADD CONSTRAINT def_contrainte;

3.4- Suppression d'une contrainte

- Syntaxe :

* Suppression de la contrainte clé primaire

ALTER TABLE nom_table

DROP PRIMARY KEY [CASCADE];

L'option cascade pour pouvoir supprimer une clé primaire réferencée.

* Suppression d'une contrainte autre que la clé primaire

ALTER TABLE nom_table

DROP CONSTRAINT nom_contrainte;

où le nom_contrainte est celui de la contrainte à supprimer.

Pour retrouver les différentes contraintes avec leur propriétés, on peut utiliser la commande suivante:

SELECT * FROM user_constraints [WHERE table_name ='nom_table'];

Le nom de la table doit être écrit en majuscule.

3- Suppression de Tables

- Syntaxe :

DROP TABLE nom_table;

- Exemple :

DROP TABLE client_mahdia;

—> La contrainte de suppression de table permet de supprimer toutes les contraintes d'integrité réferentielles qui se reflètent aux clés uniques ou primaires de la table à supprimer, et ce à travers l'option CASCADE CONSTRAINTS.

4- Renommage et contrainte de synonymes de tables

* Pour changer le nom d'une table existante la commande est :

- Syntaxe :

RENAME ancien_nom TO nouveau_nom;

* Pour créer un synonyme (second nom d'une table) :

- Syntaxe :

CREATE SYNONYM nom_synonyme FOR nom_table;

* pour supprimer un synonyme :

- Syntaxe :

DROP SYNONYM nom_synonyme:

Remarque : la suppression d'une table implique la suppresssion des synonymes correspondants.
