1- Introduction

Les applications informatiques et les utilisateurs ont conjointement subi d'importantes évolutions qualitatives et quantitatives.

Comme conséquence de la vulgarisation de l'informatique dans divers domaines et la complexité sans cesse croissante des applications à informatiser, il y a eu nécessité d'avoir une répresentation intermédiaire entre le réel perçu et ce qui va être effectivement implanté.

Cette répresentation intermédiaire vise la réduction de l'écart entre le réel à informatiser et son implantation physique finale, Et ce, en utilisant un modèle de répresentation.

Un modèle est donc une répresentation du monde réel via des formalismes graphiques ou mathématiques.

Une méthode de conception est un ensemble de modèles avec un certain nombre de démarches ou étapes qui servent à guider progressivement la transformation de l'application vers l'implantation : c'est la modélisation.

Dans ce qui suit, on va présenter un aperçu sur l'historique des modèles de données, les avantages et les inconvénients de chacun.

2- Le Modèle Hiérarchique

Les SGBDs du type hiérarchique sont apparus les premiers sur le marché. IBM a commercialisé IMS-1 "Information Management System" en 1969 sur le système 360.

La conception du Schéma dans le modèle hiérarchique dépend uniquemend des liens 1 : N (Père-Fils).

Une base de données se présente comme un arbre ordonné dont les sommets sont les entoités et les arcs sont les liens. Une entité peut avoir pluieurs fils, mais un fils ne peut avoir qu'un seul père.

Les inconvénients de ce modèle :

  • Impossible de réprésenter directement les liens maillés ( N:M plusieurs à plusieurs).
  • Si on supprime l'un des pères, la suppression de tous les fils est obligatoire. Ce qui engendre des anomalies pour les opérations de stockage ( insertion,suppression,modification).

Cet Exemple met en évidence les inconvénients des représentations hiérarchiques. Prenons le cas où un projet est rattaché à plusieurs départements ( de telle sorte que le projet A devra avoir deux pères : département X et département Y ) ce qui n'est pas pemis dans le modèle hiérarchique. On devra rajouter des arborescences avec des données redondantes avec le risque de les avoir non cohérentes. Le même problème se pose lorsqu'un employé travaille sur plusieurs projets.

4- Le Modèle Relationnel

Le modèle Relationnel a été défini par Edgar Codd en 1970 à IBM San José en Californie.

Le modèle relationnel permet:

  • Une description simple sous forme tabulaire des entités.
  • Une mise à jour de données sans anomalies de stockage.
  • Une manipulation non procédurale de données.
  • Une indépndance données/traitements

5- Le Modèle Objet

Un SGBD associe une base d'objets à un langage de programmation orienté objets. Ce dernier est utilisé pour la définition du schèma de la base, la manipulation des objets et pour la codification des applications.

Cette association langage-base d'objets permet de tirer profil aussi des avantages de la programmation orientée objets que de ceux des bases de données.
